AI Developer Tool Usability Research

Studied why developers struggled with Intel's AI development platform and identified ways to make it more accessible for users with varying technical backgrounds.

Client

Intel Corporation

Duration

10 months

Role

UX Researcher

Methods

Interviews · Usability Testing · Analytics

73%

of new users abandoned the platform within their first session

"I expected 'no-code' to mean simple drag-and-drop. Instead, I'm seeing technical terms I don't understand and too many settings upfront."
— Data Scientist, Manufacturing Analytics Team

Key Findings

  • Too Technical: 73% of new users abandoned within first session. Platform labeled "no-code" but still required deep technical knowledge.
  • Confusing Error Messages: When things failed, users saw technical error messages that didn't help them understand what went wrong or how to fix it.
  • Overwhelming First Experience: All settings shown upfront. Beginners saw 30+ options they didn't understand before completing their first task.
  • No Step-by-Step Guidance: Users expected a simple wizard for common tasks. Instead, faced a blank workspace requiring advanced technical knowledge.
  • No Helpful Defaults: Every setting required manual input. No intelligent suggestions based on what users were trying to build.

Problems → Solutions

Problem

"No-code" platform still required technical expertise

Users faced technical terms, complex settings, and confusing error messages that required deep technical knowledge.

Solution

Simplified mode with guided workflows

Created simpler interface with helpful defaults, plain-language explanations, and step-by-step guidance.

Problem

Error messages were too technical

When something went wrong, users saw technical error codes that didn't help them fix the problem.

Solution

Clear error messages with next steps

Plain explanations of what went wrong and clear steps to fix it, with links to helpful resources.

Problem

Too many settings shown at once

30+ settings displayed before users could start. No guidance on which were important for their task.

Solution

Show only essential settings first

Display 3-5 key settings with smart defaults. Advanced options available when needed.

Research Approach

  • 25+ developer interviews to understand workflows and pain points
  • 15 usability sessions observing first-time users building pipelines
  • Analytics analysis of where users got stuck and abandoned
  • Prototype validation with 12 users testing Beginner Mode concept
"This is much better. I just described what I wanted and it set everything up for me. That's what I expected from the beginning."
— Data Analyst, testing simplified design

Key Recommendations

  • Simplify First Experience

    Created simplified mode that shows only essential settings

    Reduced initial options from 30+ down to 3 key inputs with smart defaults. Advanced users can access full settings when needed.

  • Clearer Error Messages

    Replaced technical error messages with plain explanations

    Changed confusing technical errors into clear messages like "Your data has 10 columns, but you selected 8. Please select all required columns." Added quick-fix buttons for common errors.

  • Step-by-Step Guidance

    Designed guided workflow for common tasks

    Broke the process into 4 simple steps (Upload Data → Select Target → Choose Model → Review & Train). Reduced completion time from "never completed" to 8 minutes average.

  • Smart Suggestions

    Added intelligent recommendations based on user data

    System analyzes uploaded data and suggests appropriate options. Users can accept suggestions or customize. 82% of beginners successfully used suggested defaults.

  • Helpful Tips

    Added tooltips and contextual help

    Provided explanations of concepts right when users need them, with visual diagrams. Reduced user confusion from 67% to 12%.
